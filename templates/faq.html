Frequently Asked Questions

What is sample?

This is a sample template with minimal setup of poetry with fast api. go through README and start developing.

What are the main packages and libraries?
  • we are using streamlit for front end and Fast API for backend.
  • we also have versioning system, if you change version in `pyproject.toml`, it will automatically reflected in api
  • using `Box` for centralized messaged which later can be changed using i18n support.
What are method to run?

run `poetry run sample dev` for streamlit UI.

run `poetry run sample api` and /version` is one endpoint.

run `poetry run lint` to lint the code.

How it is different from others?

This is a useful to start the development with pre-defined template and best practices and define folder structure.

Which Python version is supported?

Python ≥ 3.11 is required. Using lower versions may cause dependency resolution or typing issues. Verify using python --version.

Where is the virtual environment created?

Poetry manages the virtual environment automatically. Run poetry env info to check the active environment. If virtualenvs.path is configured, it will be created in that directory.

How do I add a new dependency?

Use Poetry only. 

poetry add package-name

For dev dependencies: 

poetry add --group dev package-name

Do not manually edit pyproject.toml.

When should I regenerate poetry.lock?

Only when adding or upgrading dependencies. Use: 

poetry lock --no-cache --regenerate

Avoid unnecessary lock file changes in pull requests.

Why are environment variables not loading?

Ensure the ENV variable is set correctly: 

export ENV=development

or 

export ENV=production

Also confirm that corresponding .env.* file exists in the project root.

Why is Streamlit not reflecting changes?

Restart the Streamlit process. If caching issues occur, clear local cache or restart the shell.

Why is my commit failing?

Pre-commit hooks enforce linting, formatting, and type checks. Fix issues by running: 

poetry run lint
How do I fix a corrupted virtual environment?

Remove and recreate the environment: 

poetry env info
poetry env remove <env-name>
poetry install
How do I test the built package locally?

Build the package: 

poetry build

Create a clean virtual environment and install the wheel file from dist/.

What if pymongo is not found?

MongoDB support is installed as an optional extra dependency. Install using: 

pip install sample[mongo]

If using Poetry: 

poetry install --extras "mongo"
What is the expected folder structure?

All source code must reside inside the src/ directory. Business logic should not be written directly in UI or API entry files. Maintain separation between API, UI, services, and utilities.

Are secrets allowed inside the repository?

No. Never commit secrets. Use environment variables or external secret managers. Maintain a .env.example file for documentation purposes.
